# 策略入出场条件详解

本文档记录了 AgentK 中主要交易策略的入场与出场机制，基于 `src/modules/structure-detector/strategies` 目录的核心源码。

> [!info]
> 策略的路由与选择受到 [[策略路由]] 机制的控制。

## 策略总览

| 策略 | 核心入场条件 | 出场/止盈逻辑 |
| :--- | :--- | :--- |
| **[[策略详解-TurtleSoup\|Turtle Soup (海龟汤)]]** | 震荡状态下作为策略编排器。级联调用：Band Reversion > VWAP Bounce > SFP 模式。 | 结合区间宽度和波动率动态设置，常锚定区间边界及 VWAP。 |
| **[[策略详解-SqueezeStraddle\|Squeeze Straddle]]** | 震荡压缩 (Coiling)；符合 `narrowRange` / `convergingPattern`。 | 突破区间 High/Low，并结合 Buffer/Range Protectors 计算目标价位。 |
| **[[策略详解-StandardICT\|Standard ICT]]** | 顺势交易；H1 结构破坏 + M15 位移确认；依托 FVG 或 OB。 | 基于 ATR、流动性和波动率的动态 TP1/TP2/TP3。 |
| **Band Reversion** | 仅在 RangeMode 激活时运行；价格触及布林带外沿且 M5 反转。 | TP1: Session VWAP; TP2/TP3: 布林带中轨。不追趋势。 |

## 策略与增强器详解

### 1. [[策略详解-TurtleSoup\|Turtle Soup (海龟汤编排器)]]

> [!summary]
> 作为主要的反转/震荡策略编排器，优先执行 SFP 模式以规避小周期噪声，包含三大核心子策略。

- **Band Reversion**: 双通道均值回归。检测 BB (Bollinger Bands) 带宽压缩，价格触碰 Keltner 外沿，辅以 M5 转向实体确认。
- **VWAP Bounce**: 双 VWAP 弹跳确认。上一根 M5 触碰 VWAP 区域，当前 K 线实体收盘越过 VWAP。
- **SFP Pattern**: 关键位假突破 (Swing Failure Pattern)。寻找近期流动性 Sweep（扫损），并结合动能衰竭进行反向操作。

### 2. [[策略详解-SqueezeStraddle\|Squeeze Straddle (压缩突破)]]

专注于压缩后的突破交易，配置于 `squeeze-straddle.ts`。

- **入场**: 在盘整区的高点/低点上方/下方设置缓冲区突破订单。
- **过滤**: 高度依赖压缩指标 (`narrowRange`, `convergingPattern`, `atrContraction`) 和 Hurst 指数。并检查 ADX 确认没有处于强单边趋势。

### 3. [[策略详解-StandardICT\|Standard ICT (ICT 结构跟踪)]]

基于机构逻辑的顺势趋势跟踪策略 (`standard-ict.ts`)。

- **入场**: 极度依赖高层级 (H1) 结构破坏 (==MSS== / ==BOS==)，辅以 M15 级别的显著位移确认 (==Displacement==)。通过 FVG 评级或 Order Block 区间入场。
- **过滤**: 结合 EMA Direction Filter 及 Keltner Extreme 状态阻止错误入场。
- **出场**: 动态 TP 分布。TP1 通常为 `entry + risk`，TP2/TP3 依据 RR（盈亏比）设置。

### 4. Band Reversion Only (独立回归)

一个专门的影子通道 (Shadow Lane) 测试策略 (`band-reversion-only.ts`)，仅做均值回归。

> [!warning] 严格限制
> - BB 带宽历史 P60 压缩下，H1 偏离 Session VWAP 0.4~3.0 ATR。
> - M5 转向实体大于 0.35 ATR。
> - 入场不能越过区间中轴。

- **出场**: 固定瞄准 Session VWAP 和 布林带中轨，==严禁追击趋势延伸==。

### 5. Enhancers (策略增强与风控钩子)

在策略管线中 (`src/modules/structure-detector/strategies/enhancers/`)，所有入出场均受到以下增强器的二次校验：

- [x] **Entry Distance**: 确保入场价不至于距离基准价太远或太近。
- [x] **Momentum Priority**: 基于位移强度的动能优先级评分，顺势得分更高。
- [x] **Range Protector**: 确认震荡区间宽度足够，过滤无效的极窄区间假突破。
- [x] **Risk Metrics**: 计算最小盈亏比 (RiskRewardGuard) 并分布 TP1/TP2/TP3。
