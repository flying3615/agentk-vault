# 策略路由 (Strategy Routing)

## 概述
策略路由是 AgentK 的核心决策机制，负责根据实时市场结构、流动性及波动率动态选择交易策略 regime（`CONSOLIDATION`, `TREND` 或 `HYBRID`）。该机制旨在平衡趋势跟踪与均值回归，避免在震荡行情中追逐趋势，或在趋势中逆势操作。

## 决策核心
路由决策依赖 `StrategyRouteContext`，整合了以下维度：
- **市场结构**: 震荡状态、范围模式评估、H4 高低点识别。
- **趋势信号**: 位移强度、成交量、HTF (更高时间框架) 趋势对齐。
- **流动性分析**: 对关键水平（如前一日/周高低点）的扫描情况。
- **统计指标**: ATR (波动率) 及 Hurst 指数 (趋势/均值回归分类)。

## 九段分层决策逻辑 (`computeBaseRoute`)

代码内部分为 9 个决策节点，按顺序评估：

| 节点 | 条件 | 路由结果 |
|------|------|---------|
| **Section 1** | 高置信度盘整（HTF 已判定） | 强制 `CONSOLIDATION`；仅挤压突破时允许穿透到下游 |
| **Section 2** | Turtle 优先信号触发（`EXTREME_SWEEP` / `TREND_EXHAUSTION` / `H4_RANGE_EDGE`） | 至少 `HYBRID`，防止 TREND-only 屏蔽海龟汤 |
| **Section 3** | RangeMode 已激活 | 维持 `HYBRID`（不直接回 TREND），观察 M15 逃逸位移避免追单 |
| **Section 4** | 趋势脱离信号（非 RangeMode） | 路由至 `TREND` |
| **Section 5** | 盘整置信度 40–50（灰区） | 优先 `HYBRID`/`CONSOLIDATION`，防止震荡中被 TREND 追价反复打损 |
| **Section 6** | 盘整置信度 55–65（中等） | `HYBRID` |
| **Section 7** | 明确趋势信号 | `TREND` |
| **Section 8** | Hurst 打平局 | `NOISY` 降级到 `HYBRID`，不完全信任统计分类 |
| **Section 9** | 默认兜底 | 挤压突破至少给 `HYBRID`（不退回 `CONSOLIDATION`） |

## Regime 锁定 (`applyRegimeLock`)

> [!info] 防抖机制
> 防止路由在 TREND/HYBRID 间高频切换，引入时间锁：
> - 盘整置信度 ≥ 65 → 锁定 `CONSOLIDATION` **90 分钟**
> - 盘整置信度 ≥ 55 → 锁定 `HYBRID` **45 分钟**
> - 锁定期间：旧锁仍有效且当前基础路由回到 TREND，但盘整置信度 ≥ 50 时，优先维持旧锁路由

边界区域（置信度处于锁定阈值附近）会延续现有锁而非立即解锁，避免单根 K 线导致状态翻转。

## Turtle Soup 优先信号 (`detectTurtleSoupPrioritySignal`)

触发来源有三种，每种对应不同的市场场景：

### 1. 极值扫损 (`EXTREME_SWEEP`)
- 价格刺穿高质量流动性水平（PDH/PDL、Equal Highs/Lows、H4 Equal 等）
- 根据水平类型有基础分层：等高等低 > PDH/PDL > 普通 Swing
- 综合扫损质量评分（`sweepScore`）超过阈值时，触发 Arbiter 强制硬覆盖（+30 诊断分）

### 2. 趋势耗尽 (`TREND_EXHAUSTION`)
- 基于位移的幅度和影线比例判断当前趋势动能衰减
- 在明显的推进位移之后，随即出现显著上影/下影（弱势收盘），触发信号

### 3. H4 区间边缘 (`H4_RANGE_EDGE`)
- 检测 H4 级别的宏观区间高低点（`macroHigh` / `macroLow`）
- 价格触及边界后出现再占位（Reclaim），触发反转优先

### 4. 第三推耗尽（H4 摆动区）
- `detectThirdPushAtSwingPrioritySignal` 仅在非盘整趋势市中触发
- 牛市耗尽：价格接近 H4 摆动高点且三推完成 → 空头反转优先
- 熊市耗尽：价格接近 H4 摆动低点且三推完成 → 多头反转优先
- 最小得分阈值：55 分，且价格须在关键水平 1.0 ATR 范围内

## 趋势脱离 (`detectConsolidationTrendEscape`)

在盘整区间确认的前提下，检测位移强度是否足够大、方向是否与 HTF 一致，若满足则判断为趋势"逃离"盘整，切换至 `TREND`。缓冲比例为 0.1%（`TREND_ESCAPE_BREAK_BUFFER_RATIO`），避免噪音假突破触发。

## 统一路由入口 (`resolveStrategyRoute`)

> [!note] 建议使用此入口
> `resolveStrategyRoute` 一次性计算所有信号（`trendEscape`、`turtlePriority`），确定基础路由，应用 Regime 锁，并返回完整 `StrategyRouteDecision`。
> 不应分别调用 `determineStrategyRoute` + `applyRegimeLock` + `detect*` 函数（会导致信号重复计算）。

返回结构：
- `route`: 最终路由决策
- `baseRoute`: 未经锁修正的基础路由
- `regimeLockResult`: 锁状态详情
- `trendEscape` / `turtlePriority`: 各信号详情
