# 运维与开发指南 (Maintenance)

AgentK 提供了一套辅助脚本，用于在日常开发与部署时进行系统健康度检查和连通性测试。

## 常用脚本 (Scripts)

在 `scripts/` 目录下，包含以下重要的诊断与测试脚本：

> [!tip] 脚本执行
> 一般可以通过 `npx ts-node scripts/<脚本名称>.ts` 或配置的 npm 快捷命令运行。

- `check-balance.ts` / `check-balance-raw.ts`: 检查 Hyperliquid 主钱包和子账户的资金与保证金余额。
- `check-data-depth.ts`: 测试 `DataFetcher` 拉取的 K 线数据深度。
- `verify-connection.ts`: 验证与交易所环境（主网/测试网）的网络连通性，并检查挂单 (`Open Orders`) 状态。
- `test-qwen-api.ts`: 独立测试 Qwen API 等 LLM 接口的连通性与密钥是否正确。
- `test-mainnet-order.ts`: 主网下单测试安全脚本，会故意下发远离市价（例如市价的 50%）的限价单来验证交易 API 链路，而不造成实际成交。

## 测试与质量保证

> [!important] 单元测试规范
> AgentK 采取了极高覆盖率的单元测试。核心逻辑（如 `controller` 和 `structure-detector`）下均配有同名的 `.test.ts` 文件。

你可以使用 `vitest` 等测试框架执行，例如确保 `ai-close-cooldown`, `auto-reversal-logic` 等控制台流转在上线前不受破坏。
