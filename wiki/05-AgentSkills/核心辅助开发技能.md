# 核心辅助开发技能 (Agent Skills)

> [!abstract] 定义
> 此目录记载的 **Skill (技能)** 不是 AgentK 的运行时交易代码，而是位于开发者本地环境（`.agents/skills/` 目录）中，**专门为命令行 AI 助手（如当前操作你的助手）提供的工作流标准**。
>
> 助手在接到复杂任务时，会自动调用这些预设的技能（Superpowers），以最规范、最具纪律性的方式协助你完成软件开发、代码审查和知识库维护。

以下是适用于 AgentK 项目开发生命周期的核心技能清单：

## 1. 架构设计与规划 (Planning & Design)

- **`brainstorming` (头脑风暴)**
  - **触发场景**: 在编写任何代码、创建新组件或修改逻辑之前**必须**调用的技能。
  - **工作流**: AI 必须先探索用户意图、梳理当前项目的环境限制、给出 2-3 个备选方案并附带 Trade-offs（权衡），只有在你（人类）批准了方案后，才会进入下一步的开发。
- **`writing-plans` (编写计划)**
  - **触发场景**: 当你拥有一个多步骤任务的需求或规范，但还未触碰代码时使用。
- **`ui-ux-pro-max` (界面设计增强)**
  - **触发场景**: 前端组件（React/Vite）开发和优化。包含各类风格（如暗黑模式、玻璃拟态）、调色板和 Shadcn/ui 的深度集成指引。

## 2. 代码开发与执行 (Execution)

- **`test-driven-development` (TDD 测试驱动)**
  - **触发场景**: 在实现任何功能或修复 Bug 之前，强制要求先写测试代码。
- **`systematic-debugging` (系统化调试)**
  - **触发场景**: 当遇到 Bug、测试失败或意外行为时。AI 必须遵循系统化的排查方法，而不是盲目猜测或直接给补丁。
- **`using-git-worktrees` (工作区隔离)**
  - **触发场景**: 当开始新的功能开发或执行计划前，自动创建一个隔离的 git worktree，保证当前工作区不被污染。
- **`dispatching-parallel-agents` / `subagent-driven-development` (子智能体并行开发)**
  - **触发场景**: 面对 2 个以上互不依赖的独立任务时，AI 助手将同时派发多个子助手并发工作，极大缩短耗时。

## 3. 审查与验证 (Review & Verification)

- **`verification-before-completion` (完工前强制验证)**
  - **触发场景**: 在 AI 试图声称“任务完成”、“Bug已修复”或准备创建 PR 之前，强制要求其运行测试和 Lint 脚本，**“无证据，不邀功”**。
- **`requesting-code-review` / `receiving-code-review` (代码审查工作流)**
  - **触发场景**: 在主要功能合并前，或者收到其他开发者的 review 意见时使用。AI 被要求保持技术严谨性，而不是一味地对 review 意见无脑附和。
- **`finishing-a-development-branch` (分支收尾)**
  - **触发场景**: 代码完成、测试通过后，AI 会为你提供结构化的合并 (Merge) 或 Pull Request (PR) 选项。

## 4. 知识库与 Obsidian 管理 (Knowledge & Vault)

> [!note] 知识流转
> 我们当前构建 `wiki/` 目录正是利用了下面这套专门为 Obsidian 打造的专属技能集。

- **`obsidian-markdown`**: 生成完全符合 Obsidian 规范（如双链 `[[]]`、Callouts `> [!note]` 和 Frontmatter）的 Markdown 文件。
- **`obsidian-cli`**: 通过 CLI 与你的本地 Vault 进行交互、搜索与管理。
- **`defuddle`**: 清洗从网页上抓取的信息，去掉杂乱的导航元素，提取最纯净的 Markdown 以便吸收进知识库。
- **`codebase-reference-vault`**: 专门用于阅读陌生代码库（如 `raw/repomix-output.md`），并将其拆解为网状连接的、可搜索的代码阅读知识库（就如我们当前所做的）。
- **`json-canvas` / `obsidian-bases`**: 在 Obsidian 中以 JSON Canvas 格式创建白板或数据库视图（Base）。

## 5. 项目专有技能 (Project-Specific AgentK Skills)

AgentK 在 `../agentK-docker/.agents/skills/` 目录下维护了针对该量化系统的核心重放与回测专属技能。当你（AI）需要排查交易事故或验证策略改动时，**必须**使用这两项技能：

- **[[Skill-SegmentBacktestReplay\|segment-backtest-replay (片段回测重放)]]**
  - **触发场景**: 当你修改了策略代码（如调整了检测逻辑），需要验证它在特定历史时间段（例如昨天上午的横盘阶段）是否能正确生成入场点时。
  - **特点**: 在本地回放量化信号引擎，并自动生成 HTML 和带出入场标记的 K 线截图 (PNG) 报告。
- **[[Skill-VPSTradeReplay\|vps-trade-replay (VPS 实盘复盘工具)]]**
  - **触发场景**: 当遇到实盘中莫名其妙的亏损、错误的 AI 平仓等“黑天鹅”事故，需要调查服务器现场时。
  - **特点**: 禁止使用本地数据库，强制 AI 助理通过 SSH 拉取 VPS 上的 `aiLogs`、`decisionLogs` 和带有 `traceId` 的现场证据，支持 SQL 检索和完整的数据包 (`replay bundle`) 导出。

## 6. 元技能 (Meta)

- **`using-superpowers`**: AI 每次启动时的底层基座，要求“哪怕只有 1% 的可能会用到某个技能，也必须触发前置检查”。
- **`writing-skills`**: 当你需要我为你创造或修改一个新的 `.agents/skills` 时使用，要求验证技能有效性后再部署。
